📌 What is Linear Search?

Linear Search is the simplest searching algorithm. It sequentially checks each element of the array until a match is found or the entire array has been traversed.

  • ✅ If element is found → returns the index
  • ✅ If element is not found → returns -1 (since valid indices start at 0)
  • ✅ Works on both sorted and unsorted arrays
💡 Return -1 convention: Array indices start at 0, so -1 safely indicates "not found".
🎯 Understanding the Search Operation

Given an array and a target value, find if the target exists in the array:

  • ✅ If found → return the index of the target
  • ✅ If not found → return -1
Example: Array = [10, 20, 30, 40, 50], Target = 30 → Output: index 2
Example: Target = 80 → Output: -1
⚙️ Linear Search Implementation (Java)
public class ArrayLinearSearch {
    public static void main(String[] args) {
        // Array can be sorted or unsorted
        int[] arr = {10, 20, 30, 40, 50};
        int target = 30;
        
        int result = linearSearch(arr, target);
        
        if (result != -1) {
            System.out.println("Element found at index: " + result);
        } else {
            System.out.println("Element not found in the array");
        }
    }
    
    public static int linearSearch(int[] arr, int target) {
        // Traverse the array sequentially
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == target) {
                return i;   // Return index if found
            }
        }
        return -1;   // Not found
    }
}
🧪 Output: Element found at index: 2
📖 Step-by-Step Dry Run

Array: [10, 20, 30, 40, 50], Target = 30

  • Iteration 1: i=0 → arr[0]=10 → 10 == 30? No → continue
  • Iteration 2: i=1 → arr[1]=20 → 20 == 30? No → continue
  • Iteration 3: i=2 → arr[2]=30 → 30 == 30? Yes → return 2 ✅
✅ Found at index 2!
🔀 Works on Unsorted Arrays Too
int[] unsortedArr = {50, 10, 80, 25, 99};
int target = 25;
int position = linearSearch(unsortedArr, target);
// Output: position = 3 (found at index 3)
💡 Linear search does NOT require the array to be sorted — it works on any array!
⏱️ Time Complexity of Linear Search
  • Best Case: O(1) — element found at the first index (index 0)
  • Worst Case: O(n) — element at the last index or not present (must traverse all n elements)
  • Average Case: O(n) — on average, we scan half the array
📉 Worst-case: O(n) → for large n, inefficient compared to Binary Search (O(log n))
✅ When to Use Linear Search?
  • ✔️ The array is small (few elements)
  • ✔️ The array is unsorted and sorting is not feasible
  • ✔️ Searching in a linked list or other linear structures (no random access)
  • ✔️ Simplicity is required — easy to implement and understand
⚠️ Drawback: For large datasets (millions of elements), linear search becomes too slow. Use Binary Search for sorted arrays.

Imagine you have a stack of exam papers (unsorted). You are looking for a specific student's paper. You check each paper one by one from top to bottom until you find it. That's exactly how linear search works! ✅

🎤 Interview Questions & Answers

❓ Q1: What is Linear Search? Explain with example.

Answer: Linear search sequentially checks each element of an array until the target is found. Example: Searching for 30 in [10,20,30,40,50] checks index 0 (10), index 1 (20), then index 2 (30) → found. Returns index 2.

❓ Q2: What is the time complexity of Linear Search in worst case?

Answer: O(n) — where n is the number of elements. In worst case (element at last index or not present), we must check all n elements.

❓ Q3: Does Linear Search require the array to be sorted?

Answer: No, linear search works on both sorted and unsorted arrays. It checks each element sequentially regardless of order.

❓ Q4: What is the best case time complexity of Linear Search?

Answer: O(1) — when the target element is at the first position (index 0). We find it in the first comparison.

❓ Q5: Why does Linear Search return -1 when element is not found?

Answer: Because array indices start at 0. Returning 0 would incorrectly indicate the element is at index 0. -1 is a safe "not found" value.

❓ Q6: When would you choose Linear Search over Binary Search?

Answer: Choose Linear Search when: 1) The array is unsorted, 2) The array is very small, 3) You need simplicity, 4) Working with linked lists or other structures without random access.

📊 Linear Search vs Binary Search (Quick Preview)
  • Linear Search: Works on any array (sorted/unsorted). Time = O(n). Simple.
  • Binary Search: Requires sorted array. Time = O(log n) → much faster for large datasets.

⚠️ Edge Cases
  • Empty array: Loop doesn't run → returns -1 correctly
  • Single element array: Works fine (one comparison)
  • Duplicate elements: Returns the first occurrence (lowest index)
  • Null array: Should check for null before processing to avoid NullPointerException
📌 Summary & Key Takeaways
  • ✔️ Linear Search = sequential scan from index 0 to n-1
  • ✔️ Returns index if target found, else -1
  • ✔️ Time Complexity: O(n) worst case, O(1) best case
  • ✔️ Works on sorted and unsorted arrays
  • ✔️ Ideal for small or unsorted datasets
  • ✔️ No extra memory required → in-place algorithm
