📌 What is a Linked List?

Linked List is a linear data structure where elements are stored in nodes. Unlike arrays, linked lists do NOT store elements in contiguous memory locations. Each node contains two parts:

  • Data — the actual value stored
  • Reference (Pointer) — address of the next node
🔵 [ Data | Next ] → 🔵 [ Data | Next ] → 🔵 [ Data | null ]

Nodes are connected using pointers. The last node points to null (indicating end of list).

🏗️ Node Structure (Conceptual)
class Node {
    int data;      // Data stored in the node
    Node next;     // Reference to the next node
    
    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
💡 Each node is an object. The next pointer connects nodes together forming a chain.
⚖️ Array vs LinkedList: Key Differences

Access TimeO(1) random access via indexO(n) sequential access
Insertion/DeletionO(n) — requires shifting elementsO(1) if we have reference to node
Memory OverheadLess (only data)More (extra pointer per node)

📌 Arrays are better for frequent access by index. Linked Lists are better for frequent insertions/deletions.
📈 Dynamic Size Advantage

Arrays have fixed size. To insert/delete, we needed to create a new array. Linked Lists are dynamic — nodes can be added or removed easily without recreating the entire structure.

Insert: Simply create new node and update pointers → O(1)
Delete: Update pointers of previous node → O(1)
📚 Types of Linked Lists
1️⃣ Singly Linked List

Each node has one pointer (next) pointing to the next node. Traversal is only forward direction.

[10 | →] → [20 | →] → [30 | null]
  • Head points to first node
  • Last node points to null
  • Simple and memory efficient (only one pointer per node)
2️⃣ Doubly Linked List

Each node has two pointers: prev (previous node) and next (next node). Allows traversal in both forward and backward directions.

null ← [10 | prev/next] ↔ [20 | prev/next] ↔ [30 | prev/next] → null
  • First node's prev points to null
  • Last node's next points to null
  • Easier deletion (we know previous node)
  • Extra memory overhead (two pointers per node)
class DoublyNode {
    int data;
    DoublyNode prev;
    DoublyNode next;
    
    DoublyNode(int data) {
        this.data = data;
        this.prev = null;
        this.next = null;
    }
}
3️⃣ Circular Linked List

In a circular linked list, the last node points back to the first node, forming a circle. Can be implemented as Singly Circular or Doubly Circular.

[10 | →] → [20 | →] → [30 | →] ┐
↑_________________________________┘
  • No null references
  • Useful for round-robin scheduling, playing lists, etc.
  • Traversal continues infinitely unless we track head

🔗 Why Reference Pointer?

Since nodes are stored in non-contiguous memory (scattered across heap), we need pointers to connect them. Without pointers, we would have no way to know which node comes next.

Memory Address: 0x100 → Node: [10 | 0x200]
Memory Address: 0x200 → Node: [20 | 0x300]
Memory Address: 0x300 → Node: [30 | null]
✅ Pointers create the logical sequence even though physical memory is not continuous.
✅ Advantages & ❌ Disadvantages

Advantages:

  • ✔️ Dynamic size — grows/shrinks at runtime
  • ✔️ Efficient insertion/deletion (O(1) at head if we have reference)
  • ✔️ No memory wastage (allocates exactly as needed)
  • ✔️ Can represent complex data structures (stacks, queues, graphs)

Disadvantages:

  • ❌ No random access — must traverse from head (O(n))
  • ❌ Extra memory for pointers (8 bytes per node on 64-bit)
  • ❌ Cache unfriendly — nodes scattered in memory
  • ❌ Reverse traversal is difficult in singly linked list
🎤 Interview Questions & Answers

❓ Q1: What is a Linked List? How is it different from an Array?

Answer: Linked List is a linear data structure where elements (nodes) are stored non-contiguously, each containing data and a pointer to the next node. Arrays store elements contiguously with index-based access. Linked lists are dynamic (size can change), while arrays have fixed size. Insertion/deletion is O(1) in linked lists (with reference) vs O(n) in arrays.

❓ Q2: What are the types of Linked Lists? Explain each.

Answer: Three main types: Singly Linked List (one pointer per node, forward traversal), Doubly Linked List (two pointers per node — prev and next, bidirectional traversal), Circular Linked List (last node points back to first, no null, can be singly or doubly circular).

❓ Q3: Why is insertion/deletion faster in Linked Lists than Arrays?

Answer: In arrays, insertion/deletion requires shifting elements to maintain contiguous memory — O(n). In linked lists, we just update pointers of adjacent nodes — O(1) if we have the reference to the position. However, finding the position may still be O(n).

❓ Q4: Can we implement a Linked List in Java without using built-in LinkedList class?

Answer: Yes! We create a custom Node class (with data and next pointer) and manually manage head, insertion, deletion, and traversal. This is a common interview task.

class Node {
    int data;
    Node next;
    Node(int data) { this.data = data; }
}

❓ Q5: What is the time complexity to search an element in a Linked List?

Answer: O(n) in worst case because we must traverse from head to find the element (no random access).

❓ Q6: What happens if we lose the head reference of a Linked List?

Answer: The entire list becomes unreachable and will be garbage collected. Always store the head reference to access the list.

❓ Q7: When would you choose a Linked List over an Array?

Answer: Choose Linked List when: 1) You need frequent insertions/deletions, 2) You don't know the size in advance, 3) You don't need random access, 4) Memory is not a constraint (extra pointer overhead is acceptable).

🌍 Real-Life Applications
  • Browser History: Doubly linked list to go forward/backward
  • Music Playlist: Circular linked list for repeat mode
  • Undo/Redo Feature: Doubly linked list
  • Image Viewer: Next/Previous navigation
  • Process Scheduling: Circular linked list for round-robin
  • Hash Table Collision Handling: Chaining using linked lists
📌 Summary & Key Takeaways
  • ✔️ Linked List = nodes with data + pointer, stored non-contiguously.
  • ✔️ Dynamic size — no fixed capacity, easy insert/delete.
  • ✔️ Three types: Singly (forward), Doubly (both directions), Circular (last → first).
  • ✔️ Array vs LinkedList: Arrays have O(1) access, fixed size; Linked Lists have O(1) insert/delete (with reference), dynamic size.
  • ✔️ Trade-offs: Linked Lists use extra memory for pointers, no random access.

🎯 Interview Tip: Be ready to draw diagrams of Singly, Doubly, and Circular Linked Lists. Know the time complexities of search, insert, delete. Explain why insertion is O(1) if you have the node reference.
