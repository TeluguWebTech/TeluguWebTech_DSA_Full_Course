📌 Module 1: Understanding the Building Blocks
🔹 1. Data – The Raw Material

Data is any collection of raw, unprocessed facts — numbers, text, images, or symbols. By itself, data carries no meaning. It becomes valuable only after processing.

📦 Raw Data: "Product: Laptop | Price: 75000 | Stock: 12" → just numbers & words
🧠 After Processing: "Laptop is in stock with 12 units at ₹75,000" → Meaningful information
💡 Key Insight: Data + Processing = Information
🔹 2. Structure – Organizing Data for Efficiency

A data structure is a specialized way of organizing, storing, and accessing data so that operations (search, insert, delete) become fast and efficient.

🏢 Real-world example – Hospital Patient Records:
❌ Without structure: All records in one unsorted pile → takes 30 minutes to find a patient
✅ With structure: Sorted by ID + indexed by name → find any record in 2 seconds
📚 Library Analogy: Books arranged by genre → author → title. You find any book instantly!
💻 Programming Structures: Arrays, Linked Lists, Stacks, Queues, Trees, Hash Tables
🔹 3. Algorithm – Step-by-Step Problem Solving

An algorithm is a finite sequence of well-defined instructions to solve a specific problem. It takes input, processes it, and produces output.

🍕 Algorithm: Making a Pizza
1. Prepare dough → 2. Add sauce & cheese → 3. Add toppings → 4. Bake at 200°C for 15 min → 5. Serve hot 🍕 
// Algorithm: Find maximum number in an array
Input: [3, 8, 1, 9, 4]
Step 1: Assume first element is max = 3
Step 2: Compare with 8 → max = 8
Step 3: Compare with 1 → max stays 8
Step 4: Compare with 9 → max = 9
Step 5: Compare with 4 → max stays 9
Output: 9
⚙️ Every app, website, and software runs on algorithms behind the scenes.

🎤 Module 7: Common DSA Interview Questions (With Answers)

❓ Q1: What is the difference between an Array and a Linked List?

Answer: Array stores elements in contiguous memory (fast random access, slow insert/delete). Linked List stores nodes connected by pointers (slow random access, fast insert/delete).

❓ Q2: Explain Big O notation with an example.

Answer: Big O describes algorithm complexity. O(1) = constant (array access). O(n) = linear (loop). O(n²) = quadratic (nested loops). Example: Finding an element in unsorted array = O(n).

❓ Q3: What is the difference between Stack and Queue?

Answer: Stack = LIFO (Last In First Out) like a pile of plates. Queue = FIFO (First In First Out) like a ticket counter line.

❓ Q4: What is recursion? Give an example.

Answer: Recursion is when a function calls itself. Example: Factorial(n) = n * factorial(n-1) with base case factorial(0)=1.

❓ Q5: Which sorting algorithm is fastest? When to use which?

Answer: Quick Sort is fastest on average. Merge Sort is stable and guarantees O(n log n). Use Insertion Sort for small datasets. Use Heap Sort when memory is limited.
