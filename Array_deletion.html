📌 Three Types of Deletion Problems

In a static array (fixed-size), we cannot directly delete elements. Instead, we create a new array of size-1 and copy all elements except the one to delete. The three deletion scenarios are:

  • 1. Delete from Beginning: Remove first element (index 0)
  • 2. Delete from End: Remove last element (index length-1)
  • 3. Delete from Any Index: Remove element at specific position
⏱️ Overall time complexity for all cases: O(n) due to copying elements
📍 1. Delete from Beginning

Problem: Delete first element (10) from array [10, 20, 30, 40, 50].

Approach:

  • Create new array of size = old length - 1
  • Copy all elements from index 1 onwards to new array
  • Skip the element at index 0
public static int[] deleteFromBeginning(int[] arr) {
    if (arr == null || arr.length == 0) return arr;
    
    int[] newArr = new int[arr.length - 1];
    
    for (int i = 1; i < arr.length; i++) {
        newArr[i - 1] = arr[i];
    }
    
    return newArr;
}
Visual: [10,20,30,40,50] → create new array size 4 → [20,30,40,50]
📌 Time Complexity: O(n) (copying n-1 elements)
📍 2. Delete from End

Problem: Delete last element (50) from array [10, 20, 30, 40, 50].

public static int[] deleteFromEnd(int[] arr) {
    if (arr == null || arr.length == 0) return arr;
    
    int[] newArr = new int[arr.length - 1];
    
    for (int i = 0; i < arr.length - 1; i++) {
        newArr[i] = arr[i];
    }
    
    return newArr;
}
Visual: [10,20,30,40,50] → create new array size 4 → [10,20,30,40]
📌 Time Complexity: O(n) (copying n-1 elements)
📍 3. Delete from Any Index

Problem: Delete element at index 2 (value 30) from array [10, 20, 30, 40, 50].

public static int[] deleteFromIndex(int[] arr, int index) {
    // Validate index
    if (arr == null || arr.length == 0 || index < 0 || index >= arr.length) {
        System.out.println("Invalid index");
        return arr;
    }
    
    int[] newArr = new int[arr.length - 1];
    int j = 0;
    
    for (int i = 0; i < arr.length; i++) {
        if (i == index) {
            continue;  // Skip the element to delete
        }
        newArr[j++] = arr[i];
    }
    
    return newArr;
}
Visual: [10,20,30,40,50] → delete index 2 (30) → [10,20,40,50]
📌 Time Complexity: O(n) (copying n-1 elements)
📦 Complete Java Implementation
import java.util.Arrays;

public class ArrayDeletion {
    
    // Delete element at specific index
    public static int[] deleteFromIndex(int[] arr, int index) {
        if (arr == null || arr.length == 0 || index < 0 || index >= arr.length) {
            System.out.println("Invalid index: " + index);
            return arr;
        }
        
        int[] newArr = new int[arr.length - 1];
        int j = 0;
        
        for (int i = 0; i < arr.length; i++) {
            if (i == index) {
                continue;  // Skip this element
            }
            newArr[j++] = arr[i];
        }
        return newArr;
    }
    
    // Delete first element (index 0)
    public static int[] deleteFromBeginning(int[] arr) {
        return deleteFromIndex(arr, 0);
    }
    
    // Delete last element (index length-1)
    public static int[] deleteFromEnd(int[] arr) {
        return deleteFromIndex(arr, arr.length - 1);
    }
    
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};
        
        System.out.println("Original array: " + Arrays.toString(numbers));
        
        // Delete from beginning
        int[] afterDeleteStart = deleteFromBeginning(numbers);
        System.out.println("After deleting first element: " + Arrays.toString(afterDeleteStart));
        
        // Delete from end
        int[] afterDeleteEnd = deleteFromEnd(numbers);
        System.out.println("After deleting last element: " + Arrays.toString(afterDeleteEnd));
        
        // Delete from index 2 (value 30)
        int[] afterDeleteIndex = deleteFromIndex(numbers, 2);
        System.out.println("After deleting index 2: " + Arrays.toString(afterDeleteIndex));
    }
}
🧪 Output:
Original array: [10, 20, 30, 40, 50]
After deleting first element: [20, 30, 40, 50]
After deleting last element: [10, 20, 30, 40]
After deleting index 2: [10, 20, 40, 50]

Original Array: [10, 20, 30, 40, 50]
Delete index 2 (value 30):

Step 1: i=0 (10) → not index to delete → copy to newArr[0] = 10, j=1
Step 2: i=1 (20) → not index to delete → copy to newArr[1] = 20, j=2
Step 3: i=2 (30) → IS index to delete → SKIP (continue), j stays 2
Step 4: i=3 (40) → copy to newArr[2] = 40, j=3
Step 5: i=4 (50) → copy to newArr[3] = 50, j=4

Result: [10, 20, 40, 50] ✅
💡 The continue keyword skips copying the element to be deleted!
⏱️ Time Complexity Summary
  • Delete from Beginning: O(n) — copy remaining n-1 elements
  • Delete from End: O(n) — copy first n-1 elements
  • Delete at Index: O(n) — copy all elements except the deleted one
📌 All deletion operations in static arrays have O(n) time complexity due to element copying.
🎤 Interview Questions & Answers

❓ Q1: How do you delete an element from a static array in Java?

Answer: Since static arrays have fixed size, you must create a new array of size = original length - 1, then copy all elements except the one to delete. Use a loop and skip the target index using continue.

❓ Q2: What is the time complexity of deleting an element from an array?

Answer: O(n) — linear time. Because we must copy all remaining elements to the new array. If the array has n elements, we copy n-1 elements.

❓ Q3: Can we delete an element from an array without creating a new array?

Answer: No, because static arrays have fixed size. You can set the value to null/zero, but the size remains the same. For true deletion (size reduction), you need a new array.

❓ Q4: What happens if we try to delete from an invalid index?

Answer: We should validate the index first. If index < 0 or index >= array length, print an error message and return the original array (or throw an exception).

❓ Q5: How is deletion different from setting a value to zero?

Answer: Setting to zero keeps the element (just changes value). Deletion removes the element entirely and reduces array size.

❓ Q6: What is the best way to handle deletions if you need dynamic size?

Answer: Use ArrayList in Java, which handles dynamic resizing automatically. Internally, it creates a new array when needed.

⚠️ Edge Cases to Handle
  • Empty array: Return the same array or print error
  • Single element array: Delete operation returns empty array (size 0)
  • Invalid index (negative or ≥ length): Print error and return original array
  • Null array: Check for null before processing
📝 Delete vs Remove vs Clear
  • Delete: Remove an element and reduce array size (creates new array)
  • Set to zero/null: Element still exists, just value changed
  • Clear: Remove all elements (create empty array)
📌 Summary & Key Takeaways
  • ✔️ Static arrays have fixed size → create new array for deletion
  • ✔️ Delete from beginning: Copy from index 1 to new array
  • ✔️ Delete from end: Copy from index 0 to length-2
  • ✔️ Delete at index: Use continue to skip the target index
  • ✔️ Time complexity: O(n) for all three cases
  • ✔️ Always validate index before deletion
