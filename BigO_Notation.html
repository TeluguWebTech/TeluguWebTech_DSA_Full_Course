⏱️ 1. What is Time Complexity?

Time complexity means how fast your code runs as the input size grows. Let's say your input size is n. If n becomes 10 times bigger, how much slower would your program become? That is exactly what time complexity tells us.

⚠️ Important: Time complexity is NOT measured in seconds because different systems run at different speeds. Instead, we measure the number of operations performed.
📐 2. What is Big-O Notation?

Big-O notation is a way to represent the worst-case performance of an algorithm. It tells us how the time grows when n increases.

  • Big-O focuses on the growth rate, not on exact values.
  • For example: if you have 3n + 5 operations, we call it O(n) because constants don't matter for large inputs.
📊 Example: 5n + 1000 → Still O(n)
📊 2n² + 3n + 10 → O(n²) (n² dominates)
📊 3. Common Big-O Complexities (With Examples)
ComplexityNameDescriptionExample
O(1)Constant TimeTime does NOT change with input sizeAccessing array element: arr[5]
O(log n)Logarithmic TimeTime grows slowly — halves the input each timeBinary Search
O(n)Linear TimeTime grows linearly with input sizeLooping through an array of size n
O(n log n)Linearithmic TimeFaster than quadratic, common in efficient sortsMerge Sort, Quick Sort (average)
O(n²)Quadratic TimeTime grows exponentially — nested loopsBubble Sort, nested loops
O(2ⁿ)Exponential TimeVery slow — used in some recursive problemsRecursive Fibonacci (naive)

🔹 O(1) - Constant Time

No matter if the array size is 10 or 10 million, it takes the same time.

int x = arr[5];  // O(1) - direct access

🔹 O(n) - Linear Time

If n doubles, time also doubles.

for(int i = 0; i < n; i++) {
    System.out.println(i);
}

🔹 O(n²) - Quadratic Time

Nested loops — if n=100, operations ≈ 10,000

for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        System.out.println(i + "," + j);
    }
}

🔹 O(log n) - Logarithmic Time

Binary Search: if n = 1 million, log₂(1M) ≈ 20 steps only!

// Binary search on sorted array
// Each step halves the search space
📏 4. Simple Rules to Calculate Big-O
🎯 Rule 1: Ignore Constants
// Two separate loops (2n operations)
for(int i = 0; i < n; i++) {
    System.out.println(i);
}
for(int j = 0; j < n; j++) {
    System.out.println(j);
}
// Complexity: O(2n) → O(n) (ignore constant 2)
🎯 Rule 2: Nested Loops Multiply
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        System.out.println(i + "," + j);
    }
}
// Complexity: O(n × n) = O(n²)
🎯 Rule 3: Different Loops Add
for(int i = 0; i < n; i++) { }  // O(n)
for(int j = 0; j < m; j++) { }  // O(m)
// Complexity: O(n + m) (if n and m are different)
🎯 Rule 4: Drop Smaller Terms
O(n² + n) → O(n²) because n² dominates
O(2ⁿ + n²) → O(2ⁿ) exponential dominates
💻 5. Practical Examples
Example 1: Single Loop
for(int i = 0; i < n; i++) {
    System.out.println("Hello");
}
// Complexity: O(n)
Example 2: Nested Loops
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        System.out.println(i);
    }
}
// Complexity: O(n²)
Example 3: Logarithmic (i doubles each time)
int i = 1;
while(i < n) {
    System.out.println(i);
    i = i * 2;  // i doubles each time
}
// Complexity: O(log n) because i grows exponentially
💾 6. Space Complexity (Extra Memory)

Space complexity means the extra memory used by your program. When writing code for large applications, both time and space complexity are crucial.

// Example 1: O(n) space - array of size n
int[] arr = new int[n];  // O(n) extra space

// Example 2: O(1) space - fixed number of variables
int a = 10;
int b = 20;
int c = a + b;  // O(1) - constant space
💡 Good developers optimize BOTH time AND space. Sometimes we trade space for faster time (caching, memoization).
🎤 7. Why Interviewers Care About Big-O

In interviews, it's not enough that your code works. Interviewers want to know if the solution is optimized or not.

  • ❌ Brute force solution might be O(n²)
  • ✅ Optimized solution might be O(n) or O(log n)
  • 🏢 Companies like Google, Amazon, Microsoft always prefer the best possible complexity

🎯 8. Interview Questions & Answers on Time Complexity & Big-O

❓ Q1: What is Big-O notation? Why is it important?

Answer: Big-O notation describes the worst-case time complexity of an algorithm. It tells us how the runtime grows as the input size (n) increases. It's important because it helps us compare algorithm efficiency independently of hardware. Companies use Big-O to evaluate if a solution will scale for large inputs (like millions of users).

❓ Q2: What is the time complexity of accessing an element in an array?

Answer: O(1) — constant time. No matter how large the array is, accessing an element by index (like arr[5]) takes the same amount of time.

int element = arr[1000];  // O(1) even for million-sized array

❓ Q3: What is the complexity of a simple for loop from 1 to n?

Answer: O(n) — linear time. If n = 1000, loop runs 1000 times. If n = 2000, loop runs 2000 times. Time doubles when n doubles.

❓ Q4: What is the time complexity of nested loops (both running n times)?

Answer: O(n²) — quadratic time. For n = 1000, operations = 1,000,000. For n = 2000, operations = 4,000,000 (4x slower). This is common in bubble sort and brute-force algorithms.

❓ Q5: Why is O(log n) considered very efficient?

Answer: Because the number of operations grows very slowly. In binary search, each step halves the search space. For n = 1 million, log₂(1,000,000) ≈ only 20 operations. Increasing n to 2 million only adds 1 more operation!

📊 Example: Binary search on sorted array of size 1 million → ~20 comparisons.
Linear search on same array → up to 1 million comparisons.

❓ Q6: What is the difference between Time Complexity and Space Complexity?

Answer: Time complexity measures how fast an algorithm runs (number of operations). Space complexity measures how much extra memory an algorithm uses. Sometimes we trade space for time (e.g., caching, memoization) to make algorithms faster.

❓ Q7: What is the complexity of this code?

int sum = 0;
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        sum += i * j;
    }
}
for(int k = 0; k < n; k++) {
    System.out.println(k);
}

Answer: O(n² + n) → O(n²) (drop smaller term n). The nested loop dominates the runtime.

❓ Q8: Can an algorithm have O(1) time complexity? Give an example.

Answer: Yes. Examples include: accessing array element by index, inserting at the end of a dynamic array (amortized), hash table lookup (average case), and mathematical formulas like sum = n * (n+1) / 2.

❓ Q9: What is the complexity of recursive Fibonacci (naive implementation)?

int fib(int n) {
    if(n <= 1) return n;
    return fib(n-1) + fib(n-2);
}

Answer: O(2ⁿ) — exponential time. Very slow for n > 40. Optimized version (DP) gives O(n). This is why interviewers ask for optimization.

❓ Q10: How would you optimize an O(n²) solution to O(n log n) or O(n)?

Answer: Common techniques include:

  • Using sorting + two pointers instead of nested loops
  • Using HashMaps for O(1) lookups
  • Using Binary Search instead of linear search
  • Using Divide and Conquer (Merge Sort, Quick Sort)
Example: Finding a pair with given sum — brute force O(n²), using HashMap O(n).

  • ✅ Time Complexity = How execution time grows with input size
  • ✅ Big-O Notation = Worst-case performance representation
  • ✅ O(1) → Constant (array access)

  • ✅ O(n) → Linear (single loop)
  • ✅ O(n²) → Quadratic (nested loops)
