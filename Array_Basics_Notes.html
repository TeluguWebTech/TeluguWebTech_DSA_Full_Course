📌 What is an Array?

Array is a data structure that stores multiple elements (collection of elements) at continuous memory locations. Elements are stored side by side in memory with no gaps.

Memory: [10] [20] [30] [40] [50] → contiguous (no gaps)
  • ✅ Stores homogeneous data (same data type)
  • ✅ Fixed size (static array)
  • ✅ Random access using index
🔢 Indexing Concept

Indexing assigns a unique ID (index) to each element. Array indexing always starts from 0 (base index = 0).

Index: 0 1 2 3 4
Value: 10 20 30 40 50
💡 Interview Tip: Base index is 0. Last index = length - 1.
⚖️ Static vs Dynamic Arrays
  • Static Array: Fixed size, specified at creation time. Cannot grow or shrink.
  • Dynamic Array: Size can grow/shrink at runtime (e.g., ArrayList in Java).
🏗️ Indexing & Memory Layout
Index: 0 1 2 3 4
Arr: 10 20 30 40 50
Memory: [0x100] [0x104] [0x108] [0x10C] [0x110] (contiguous)
  • First element index = 0
  • Second element index = 1
  • Index increments by 1 for each subsequent element
  • Memory layout is continuous
⏱️ Time Complexity of Access

O(1) — Constant Time - The biggest advantage of arrays!

  • Access any element directly using its index
  • No traversal needed
  • Same time whether array has 10 elements or 10 million elements
📌 Access Time Complexity: O(1) - Best possible!
💻 Java Implementation
public class ArrayBasics {
    public static void main(String[] args) {
        // Method 1: Declaration + Initialization (separate lines)
        int[] numbers;
        numbers = new int[5];
        
        // Method 2: Declaration + Initialization (single line)
        int[] numbers2 = new int[5];
        
        // Inserting values
        numbers2[0] = 10;
        numbers2[1] = 20;
        numbers2[2] = 30;
        numbers2[3] = 40;
        numbers2[4] = 50;
        
        // Accessing elements
        System.out.println("Element at index 2 is: " + numbers2[2]);  // 30
        System.out.println("Element at index 4 is: " + numbers2[4]);  // 50
        
        // Method 3: Declaration + Initialization + Values (one line)
        int[] numbers3 = {10, 20, 30, 40, 50};
        System.out.println("Element at index 4 is: " + numbers3[4]);  // 50
    }
}
🧪 Output:
Element at index 2 is: 30
Element at index 4 is: 50
Element at index 4 is: 50
🔧 Three Ways to Create Arrays
// Way 1: Declaration then initialization
int[] arr1;           // declaration
arr1 = new int[5];    // initialization (size 5)

// Way 2: Declaration + initialization in one line
int[] arr2 = new int[5];

// Way 3: Direct initialization with values (size auto-detected)
int[] arr3 = {10, 20, 30, 40, 50};  // Java auto-detects size = 5
⚠️ Important: Index vs Length
Array of length 5: indices 0, 1, 2, 3, 4
Length = 5, Last index = length - 1 = 4
💡 Remember: Length starts from 1, Index starts from 0. Beginners often get confused about this!
🎤 Interview Questions & Answers

❓ Q1: Where does array indexing start in most programming languages?

Answer: Array indexing starts from 0 (zero-based indexing). The first element is at index 0, second at index 1, and so on. The last element is at index (length - 1).

❓ Q2: What is the difference between static and dynamic arrays?

Answer: Static arrays have a fixed size determined at creation time and cannot change. Dynamic arrays can grow or shrink at runtime (e.g., ArrayList in Java).

❓ Q3: What is the time complexity to access an element in an array?

Answer: O(1) - constant time. Because arrays use contiguous memory and indexing, we can directly access any element using its index without traversal.

❓ Q4: What is contiguous memory allocation?

Answer: Contiguous memory allocation means elements are stored in adjacent/sequential memory locations with no gaps between them. Arrays use contiguous memory, which enables O(1) access.

❓ Q5: What is the base index of an array?

Answer: The base index is the index of the first element, which is always 0 in zero-based indexing languages like Java, C++, Python, and JavaScript.

❓ Q6: Can we change the size of a static array after creation?

Answer: No, static arrays have fixed size. To change size, we must create a new array and copy elements (which is what dynamic arrays do internally).

📌 Summary & Key Takeaways
  • ✔️ Array = Collection of elements stored in continuous memory
  • ✔️ Indexing starts from 0 (base index = 0)
  • ✔️ Static array = fixed size; Dynamic array = growable size
  • ✔️ Memory layout = continuous (elements stored side by side)
  • ✔️ Access time complexity = O(1) - constant time
  • ✔️ Last index = length - 1
