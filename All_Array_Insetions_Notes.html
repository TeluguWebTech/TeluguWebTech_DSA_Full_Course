📌 Three Types of Insertion Problems

In a static array (fixed-size), we cannot directly insert elements. Instead, we create a new array of size+1 and shift elements. The three insertion scenarios are:

  • 1. Insert at Beginning: Add element at index 0 → shift all elements right
  • 2. Insert at End: Add element at last position
  • 3. Insert at Any Index: Insert at specific position (e.g., middle)
⏱️ Overall time complexity for all cases: O(n) due to shifting/copying elements
📍 1. Insert at Beginning

Problem: Insert value 5 at the beginning of array [10, 20, 30].

Approach:

  • Create new array of size = old length + 1
  • Insert new value at index 0
  • Copy all old elements to new array starting from index 1
public static void insertAtBeginning() {
    int[] arr = {10, 20, 30};
    int value = 5;
    
    // Create new array with size + 1
    int[] newArr = new int[arr.length + 1];
    
    // Insert new value at index 0
    newArr[0] = value;
    
    // Copy old elements to new array
    for (int i = 0; i < arr.length; i++) {
        newArr[i + 1] = arr[i];
    }
    
    System.out.println(Arrays.toString(newArr));  // [5, 10, 20, 30]
}
Visual: [10,20,30] → create new array of size 4 → [5,10,20,30]
📌 Time Complexity: O(1) for insertion + O(n) for copy = O(n)
📍 2. Insert at End

Problem: Insert value 5 at the end of array [10, 20, 30].

public static void insertAtEnd() {
    int[] arr = {10, 20, 30};
    int value = 5;
    
    // Create new array with size + 1
    int[] newArr = new int[arr.length + 1];
    
    // Copy all old elements to new array
    for (int i = 0; i < arr.length; i++) {
        newArr[i] = arr[i];
    }
    
    // Insert new value at last index
    newArr[arr.length] = value;
    
    System.out.println(Arrays.toString(newArr));  // [10, 20, 30, 5]
}
Visual: [10,20,30] → create new array of size 4 → [10,20,30,5]
📌 Time Complexity: O(n) (copying elements)
📍 3. Insert at Any Index (e.g., Middle)

Problem: Insert value 5 at index 1 (between 10 and 20) in array [10, 20, 30].

public static void insertAtIndex() {
    int[] arr = {10, 20, 30};
    int value = 5;
    int index = 1;  // Insert at position 1 (0-based)
    
    // Create new array
    int[] newArr = new int[arr.length + 1];
    
    // Copy elements before insertion point
    for (int i = 0; i < index; i++) {
        newArr[i] = arr[i];
    }
    
    // Insert new value
    newArr[index] = value;
    
    // Copy remaining elements (shift right)
    for (int i = index; i < arr.length; i++) {
        newArr[i + 1] = arr[i];
    }
    
    System.out.println(Arrays.toString(newArr));  // [10, 5, 20, 30]
}
Visual: [10,20,30] → insert 5 at index 1 → [10,5,20,30]
📌 Time Complexity: O(n) (shifting elements)

📦 Complete Java Implementation
import java.util.Arrays;

public class ArrayInsertion {
    public static void main(String[] args) {
        int[] arr = {10, 20, 30};
        
        // Insert at beginning
        int[] result1 = insertAtBeginning(arr, 5);
        System.out.println("Insert at beginning: " + Arrays.toString(result1));
        
        // Insert at end
        int[] result2 = insertAtEnd(arr, 5);
        System.out.println("Insert at end: " + Arrays.toString(result2));
        
        // Insert at index 1
        int[] result3 = insertAtIndex(arr, 1, 5);
        System.out.println("Insert at index 1: " + Arrays.toString(result3));
    }
    
    public static int[] insertAtBeginning(int[] arr, int value) {
        int[] newArr = new int[arr.length + 1];
        newArr[0] = value;
        for (int i = 0; i < arr.length; i++) {
            newArr[i + 1] = arr[i];
        }
        return newArr;
    }
    
    public static int[] insertAtEnd(int[] arr, int value) {
        int[] newArr = new int[arr.length + 1];
        for (int i = 0; i < arr.length; i++) {
            newArr[i] = arr[i];
        }
        newArr[arr.length] = value;
        return newArr;
    }
    
    public static int[] insertAtIndex(int[] arr, int index, int value) {
        if (index < 0 || index > arr.length) {
            throw new IllegalArgumentException("Invalid index");
        }
        int[] newArr = new int[arr.length + 1];
        
        // Copy elements before index
        for (int i = 0; i < index; i++) {
            newArr[i] = arr[i];
        }
        
        // Insert new value
        newArr[index] = value;
        
        // Copy remaining elements
        for (int i = index; i < arr.length; i++) {
            newArr[i + 1] = arr[i];
        }
        return newArr;
    }
}

Insert at index 1: [10, 5, 20, 30]
⏱️ Time Complexity Summary
  • Insert at Beginning: O(1) for value insertion + O(n) for copying = O(n)
  • Insert at End: O(n) for copying elements
  • Insert at Index: O(n) for shifting/copying elements
📌 All insertion operations in static arrays have O(n) time complexity due to element shifting/copying.
🎤 Interview Questions & Answers

❓ Q1: How do you insert an element at the beginning of a static array?

Answer: Create a new array of size = original length + 1. Insert the new element at index 0, then copy all original elements into the new array starting from index 1. Time complexity O(n).

❓ Q2: Why can't we directly insert into a static array?

Answer: Static arrays have fixed size. Once created, their size cannot change. To insert an element, we must create a new larger array and copy elements.

❓ Q3: What is the time complexity of inserting at the end of a static array?

Answer: O(n) because we must copy all n elements from the original array to the new array.

❓ Q4: How would you insert an element at a specific index in an array?

Answer: Create a new array of size+1. Copy elements before the index, insert the new element, then copy the remaining elements shifted right by one position.

❓ Q5: What is the difference between static and dynamic arrays regarding insertion?

Answer: Static arrays require creating a new array for each insertion (O(n)). Dynamic arrays (like ArrayList) handle resizing automatically and have amortized O(1) insertion at end.

❓ Q6: Can we insert an element at the beginning in O(1) time?

Answer: Not in a static array. In a linked list, insertion at beginning is O(1). In a dynamic array (ArrayList), insertion at beginning is O(n) because elements must be shifted.

⚠️ Important Notes
  • ✅ Always use Arrays.toString() to print arrays in human-readable format
  • ✅ Static arrays cannot be resized — we must create a new array
  • ✅ Insertion at beginning or middle requires shifting elements
  • ✅ For frequent insertions, consider using ArrayList (dynamic array)
📌 Summary & Key Takeaways
  • ✔️ Static arrays have fixed size → create new array for insertion
  • ✔️ Insert at beginning: New element at index 0, shift others right
  • ✔️ Insert at end: Copy all elements, add new element at last index
  • ✔️ Insert at index: Copy before → insert → copy remaining with +1 shift
  • ✔️ Time complexity: O(n) for all three cases (due to copying/shifting)
