🎯 Three Types of Insertion Problems

In a static array (fixed-size), we cannot directly grow the array. Instead, we create a new array with size+1 and shift elements. Understanding these three cases will boost your problem-solving & logic building.

🔹 1. Insert at Beginning

Add a new element at index 0 → shift all existing elements right.

🔹 2. Insert at End

Add element at the last position (old length index).

🔹 3. Insert at Any Index

Insert at specific position (e.g., middle) after shifting elements to the right.

⏱️ Overall time complexity for all cases: O(n) due to shifting/copying up to n elements.
📍 1. Insertion at the Beginning

We have an initial array {10, 20, 30} and we want to insert 5 at index 0. Steps:

  • Create a new array of length = original length + 1.
  • Place the new value at newArr[0].
  • Copy all old elements into newArr starting from index 1.
int[] arr = {10, 20, 30};
int value = 5;

int[] newArr = new int[arr.length + 1];   // size = 4
newArr[0] = value;                        // insert at beginning

for (int i = 0; i < arr.length; i++) {
    newArr[i + 1] = arr[i];              // shift existing elements
}

System.out.println(Arrays.toString(newArr));  // [5, 10, 20, 30]
✅ Output: [5, 10, 20, 30]
⚡ Time complexity: O(1) for assignment + O(n) for copy → O(n).
📍 2. Insertion at the End

Inserting a new element after the last existing element. The last index of new array = old array length (since zero‑based).

int[] arr = {10, 20, 30};
int value = 5;

int[] newArr = new int[arr.length + 1];

// copy all elements to the same indices
for (int i = 0; i < arr.length; i++) {
    newArr[i] = arr[i];
}
// insert at the last position
newArr[arr.length] = value;

System.out.println(Arrays.toString(newArr));  // [10, 20, 30, 5]
📤 Output: [10, 20, 30, 5]
🔁 Complexity: O(n) – because we copy all n elements.
📍 3. Insertion at Any Index (e.g., Middle)

Insert 5 at index 1 (between 10 and 20). We split the copy into two parts: elements before index, then the new value, then the remaining elements shifted right.

int[] arr = {10, 20, 30};
int value = 5;
int index = 1;   // target position (0-based)

int[] newArr = new int[arr.length + 1];

// 1. copy elements before the insertion point
for (int i = 0; i < index; i++) {
    newArr[i] = arr[i];
}

// 2. insert the new value
newArr[index] = value;

// 3. copy remaining elements (shift one position right)
for (int i = index; i < arr.length; i++) {
    newArr[i + 1] = arr[i];
}

System.out.println(Arrays.toString(newArr));  // [10, 5, 20, 30]
🧪 Result: [10, 5, 20, 30]
🧠 Generalization: Works for any valid index (0 to arr.length).
📚 Example: Larger Array (size 4 → 5)

Insert 5 at index 2 (between 20 and 30) in {10, 20, 30, 40}.

int[] arr2 = {10, 20, 30, 40};
int value2 = 5;
int index2 = 2;

int[] newArr2 = new int[arr2.length + 1];   // size 5

// copy before index (0,1)
for (int i = 0; i < index2; i++) {
    newArr2[i] = arr2[i];      // newArr2[0]=10, newArr2[1]=20
}

// place value
newArr2[index2] = value2;      // newArr2[2] = 5

// shift rest: old index2 -> new index3, old index3 -> new index4
for (int i = index2; i < arr2.length; i++) {
    newArr2[i + 1] = arr2[i];  // newArr2[3]=30, newArr2[4]=40
}

System.out.println(Arrays.toString(newArr2));
💻 Output: [10, 20, 5, 30, 40]
⏲️ Time Complexity for any-index insertion: O(n) (worst-case shift up to n elements).
📝 Summary & Key Takeaways
  • ✔️ Static arrays cannot be resized → always create a new array with length+1 for insertion.
  • ✔️ Three fundamental insertion scenarios: beginning, end, any position.
  • ✔️ Shift logic is crucial: elements after insertion point move one index right.
  • ✔️ Time complexity for all insertion variants: O(n) (linear time) due to copying/shifting.
  • ✔️ Use Arrays.toString() to easily print array contents.
  • ✔️ Mastering these patterns builds a strong foundation for dynamic arrays (ArrayList) and advanced data structures.

🎯 Interview Tip: Always remember that insertion in a static array is not in-place; you must allocate new memory. The trade-off between static & dynamic arrays often appears in system design rounds.
